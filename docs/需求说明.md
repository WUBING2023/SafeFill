# 需求说明

## 1. 项目背景

在高校、科研院所和行政单位中，老师、科研人员、行政工作人员每年需要反复填写大量表格，包括但不限于：

- 项目申报表（国自然、省自然、社科基金等）
- 人员信息表（年度考核、职称评审、人才项目）
- 成果登记表（论文、专利、获奖）
- 中期检查表、结题报告表
- 各类审批表和备案表

这些表格存在大量重复字段（姓名、身份证号、手机号、工作单位、学历、职称、项目经历等），每次都需要手工复制粘贴或重新输入，极其耗时且容易出错。同时，表格中往往包含大量个人隐私信息，使用在线填表工具或云服务存在数据泄露风险。

本项目旨在解决这一痛点：在**完全本地化、默认离线**的环境中，帮助用户从已确认的资料库中自动填写重复字段，大幅减少重复劳动，同时严格保护隐私安全。

## 2. 项目目标

1. **减少重复填表工作**：对表格中的重复字段实现自动匹配和填写，用户只需检查和补充未知信息。
2. **保留用户原始文件**：原始表格文件永不修改，所有草稿和最终文件均输出到独立目录。
3. **自动生成填写草稿**：将资料库中的信息按字段名匹配填入新表格。
4. **让用户检查确认**：通过本地网页界面，用户逐项核对填写结果，纠正匹配错误，补充未匹配字段。
5. **最终导出符合原格式的文件**：确认无误后，按原始表格格式（.docx / .xlsx）导出，保留版式和样式。

## 3. 第一版支持范围

### 3.1 支持的文件格式

| 格式 | 支持状态 | 说明 |
|------|----------|------|
| `.docx` | ✅ 第一版支持 | Word 文档中的表格 |
| `.xlsx` | ✅ 第一版支持 | Excel 表格 |
| `.pdf`（可编辑表单） | ❌ 暂不支持 | 计划第二版支持 |
| `.pdf`（扫描件） | ❌ 暂不支持 | 计划第三版支持 |
| 图片表格（JPG/PNG 等） | ❌ 暂不支持 | 计划第三版支持 |

### 3.2 第一版支持的功能

- 单一用户资料库的创建和管理
- 从 `.docx` 和 `.xlsx` 文件中提取表格内容
- 基于字段名称的模糊匹配
- 生成填写草稿（`draft_outputs`）
- 本地 HTML 页面检查填写结果
- 用户补充未知字段
- 按原格式导出最终文件（`final_outputs`）

### 3.3 第一版明确不支持的功能

- 自动提交网页申报系统
- PDF 表单处理
- 扫描件 / 图片 OCR 识别
- 多用户资料库
- API 自动调用（API 功能默认关闭，需用户手动启用）
- 自动联网搜索个人信息

## 4. 第二版计划

- 支持可编辑 PDF 表单（AcroForm、XFA）
- 支持更复杂的 Word 表格（嵌套表格、合并单元格）
- 支持多个用户资料库（如多位老师共用同一工具）
- 更智能的字段匹配算法（支持同义词、近义词匹配）

## 5. 第三版计划

- 支持扫描 PDF（通过 OCR 识别表格结构和文字）
- 支持图片表格（JPG、PNG 等格式的表格图片）
- 支持 OCR 文字识别
- 支持更复杂的版式还原（保留原表格的字体、颜色、边框等样式）
- 支持表格结构的自动识别和重建

## 6. 明确不做的事情

以下事项在任何版本中均不做，除非经过完整的安全评审并作为可选、默认关闭的功能提供：

1. **不自动联网搜索个人信息**：不使用搜索引擎、学术数据库、社交网络等查漏补缺用户信息。
2. **不自动提交申报系统**：不模拟用户在网页申报系统中自动提交表单，不接管用户的申报流程。
3. **不自动修改用户原始文件**：原始文件一经导入即为只读，所有修改操作针对副本。
4. **不偷偷上传完整资料**：即使 API 功能开启，也禁止将完整资料库或完整原始表格发送给远程服务。
5. **不替用户决定敏感信息是否保存**：任何信息保存到资料库的行为必须经过用户逐项确认。
6. **不把补充信息默认写入资料库**：用户在填写草稿中补充的未知信息，默认不保存到资料库。只有用户明确勾选"保存到资料库"的字段才会被写入。
