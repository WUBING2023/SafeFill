# 第 2 阶段使用说明 — 新表格自动填写草稿

> 本文档面向不会代码的用户。请按步骤操作，不需要任何编程知识。

---

## 本阶段做什么？

你已经完成了第 1 阶段，本地资料库（`vault\profiles\person_001.json`）中已经保存了你确认过的个人信息。本阶段的任务是：**把新的空白表格交给工具，工具自动用资料库中的信息帮你填写**，生成一份草稿和一份填写报告。

**本阶段只生成草稿，不是最终版。** 下一阶段会提供网页界面让你逐项检查。

---

## 前提条件

在开始之前，必须确保以下文件存在：

```
D:\SafeFill\vault\profiles\person_001.json
```

如果这个文件不存在，说明你还没有完成第 1 阶段。请先查阅 `docs\第1阶段使用说明.md`，完成资料库初始化后再回来。

---

## 第 1 步：准备新表格

把需要填写的**空白或半空白**表格文件（Word 或 Excel）复制到：

```
D:\SafeFill\new_forms\
```

### 支持的文件格式

| 格式 | 说明 | 示例 |
|------|------|------|
| `.docx` | Word 文档（含有表格） | 2026年度项目申报表.docx |
| `.xlsx` | Excel 表格 | 人员信息登记表.xlsx |

### 不支持的文件格式

- ❌ `.pdf` — 本阶段不支持，后续版本会支持
- ❌ `.doc` — 旧版 Word 格式，请先在 Word 中另存为 `.docx`
- ❌ `.xls` — 旧版 Excel 格式，请先在 Excel 中另存为 `.xlsx`
- ❌ 扫描件、图片 — 不支持

### ⚠️ 重要提醒

- 放入 `new_forms` 的是**副本**，工具不会修改你的原始文件。
- 可以同时放入多个表格文件，工具会逐个处理。

---

## 第 2 步：运行自动填写

### 如何运行

打开命令提示符（`cmd` 或 `PowerShell`），输入：

```
python D:\SafeFill\app\fill_form_draft.py
```

### 运行后会发生什么

1. 工具检查资料库 `person_001.json` 是否存在且包含已确认的信息。
2. 工具扫描 `new_forms` 中的所有 `.docx` 和 `.xlsx` 文件。
3. 对每个新表格：
   - 复制一份作为草稿（**不修改原始文件**）
   - 逐个检查表格中的字段名（如"姓名"、"工作单位"等）
   - 如果资料库中有匹配的信息，自动填入对应的空白单元格
   - 如果单元格已有内容，**不覆盖**，记录到报告中
   - 标记需要用户复核的字段（如项目经历、个人简介等长文本）
4. 生成**草稿文件**和**填写报告**。

## 第 3 步：查看结果

### 草稿文件

位置：
```
D:\SafeFill\draft_outputs\
```

文件名格式：
```
原文件名_自动填写草稿.docx
原文件名_自动填写草稿.xlsx
```

打开草稿文件，可以看到工具自动填写的内容。草稿尽量保留了原始格式（字体、边框、合并单元格等）。

### 填写报告

位置：
```
D:\SafeFill\filling_reports\
```

每个表格会生成两份报告：
- `filling_report_时间戳.md` — **Markdown 报告**（推荐先看这个，敏感信息已脱敏）
- `filling_report_时间戳.json` — **JSON 数据报告**（程序用）

### 报告包含什么

| 内容 | 说明 |
|------|------|
| ✅ 已填写字段 | 工具成功匹配并填写的字段 |
| ⚠️ 需用户复核 | 已填写但建议你仔细检查的字段（长文本、项目经历等） |
| ❌ 未填写字段 | 工具识别到字段名但资料库中没有对应信息 |
| 🔒 已有内容未覆盖 | 单元格已有内容，工具保留了原内容 |
| 📊 统计 | 总共处理了多少字段，填了多少 |

### 报告中的隐私保护

填写报告 Markdown 中：
- 身份证号显示为 `440***********1234`
- 手机号显示为 `138****5678`
- 通讯地址只显示前 6 个字，后面用 `***`

## 常见问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| "person_001.json 不存在" | 没完成第 1 阶段 | 先完成资料库初始化 |
| "new_forms 中没有文件" | 没放新表格 | 把待填写的 .docx / .xlsx 放入 new_forms |
| 某个字段没填上 | 表格中字段名与工具识别的名称不匹配 | 在第 3 阶段手动补充 |
| 填写的内容不对 | 字段匹配可能有歧义 | 在第 3 阶段手动修改 |
| 已有内容被覆盖否？ | 不会 | 工具不覆盖已有内容 |
| 合并单元格被破坏？ | 不会 | 工具会跳过复杂合并结构 |

## 目录说明

| 目录 | 本阶段用途 |
|------|-----------|
| `new_forms\` | **放新表格**的地方（只放副本） |
| `draft_outputs\` | 自动填写的**草稿文件** |
| `filling_reports\` | 自动填写的**报告文件** |
| `vault\profiles\` | 资料库（只读取，不修改） |
| `logs\` | 操作日志 |
| `app\` | 程序代码 |

---

## 安全说明

本阶段**完全离线运行**：

- ❌ 不会联网
- ❌ 不会调用 API
- ❌ 不会修改你的原始文件（只读取 new_forms 中的文件，在 draft_outputs 生成草稿）
- ❌ 不会修改你的资料库（只读取 vault\profiles\）
- ❌ 不会自动保存新信息
- ❌ 不会覆盖已有单元格内容
- ❌ 不会读取项目目录之外的私人文件
