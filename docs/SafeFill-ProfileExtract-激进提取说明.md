# SafeFill-ProfileExtract 激进提取说明

## 目标

让旧资料提取更积极，尽量把可用个人信息放进候选列表，减少后续填表时 vault 缺字段的问题。

## 已增强内容

1. 表格中的自定义字段也会提取：
   - 民族
   - 政治面貌
   - 导师
   - 学号
   - 爱好
   - 籍贯
   - 健康状况
   - 紧急联系人等

2. 无字段标签但格式很强的信息会作为候选：
   - 邮箱
   - 手机号
   - 身份证号
   - 由身份证号推断的出生日期

3. 这些无标签字段默认标记为 medium，需要用户确认后才保存。

## 安全边界

激进提取不等于自动保存。

- ProfileExtract 只生成候选，不写入 vault。
- ProfileSave 必须经过用户确认才会写入 `vault\profile.json`。
- 中低置信度字段不能静默覆盖 vault。
- 课程表字段仍然会被拦截为“不建议保存”。

## 对培养计划测试的结论

`示例研究生培养计划.docx` 中可提取的个人资料有限：

- 姓名
- 专业
- 学院/部门
- 研究方向
- 学号
- 导师

该文件没有手机号、邮箱、身份证号、工作单位、职称、学历、通讯地址等信息，所以无法凭空提取。

如果把培养计划当作“待填写新表”，大多数表格格子本身已有内容或是课程表字段，SafeFill 不会覆盖已有内容，因此可能出现“填入 0 个字段”。这是安全行为，不是保存失败。

## 下一步优化方向

如果还想进一步提升填表成功率，应该增强 SafeFill-FormReview，而不是只增强 ProfileExtract：

1. 支持“字段在上一行，空格在下一行”的填写。
2. 支持“姓名：____”这种同单元格冒号后填写。
3. 支持复杂表格中“字段和值跨合并单元格”的识别。
4. 所有激进填写都必须进入网页让用户确认，不直接导出最终版。
