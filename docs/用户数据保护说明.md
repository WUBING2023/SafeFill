# 用户数据保护说明

> 本文档说明项目中哪些目录和文件包含你的个人敏感数据，以及如何保护它们。

---

## 一、敏感数据存储位置

### 最高敏感级别（包含完整身份证号、手机号、地址等）

| 目录 | 包含内容 | 文件格式 |
|------|----------|----------|
| `vault\profiles\` | 你确认保存的个人资料 | `person_*.json` |
| `candidate_reviews\` | 从旧表格提取的候选信息 | `candidate_review_*.json`、`confirmed_profile.json` |

这些目录中的 `.json` 文件包含完整未脱敏的：
- 身份证号
- 手机号
- 通讯地址
- 邮箱
- 个人简介
- 项目经历

### 高敏感级别（可能包含完整个人信息）

| 目录 | 包含内容 |
|------|----------|
| `input_forms\` | 你导入的旧表格副本 |
| `new_forms\` | 你导入的新表格副本 |
| `review_results\` | 网页检查确认结果 |
| `final_outputs\` | 最终填好的表格文件 |
| `api_previews\` | 历史 API preview（已废弃 preview→send 流程），可由 Cleaner 清理 |

### 中敏感级别（应脱敏但需注意）

| 目录 | 包含内容 |
|------|----------|
| `draft_outputs\` | 自动填写草稿（含完整填写内容） |
| `filling_reports\` | 填写报告（.md 已脱敏，.json 完整） |
| `final_reports\` | 导出报告（.md 已脱敏，.json 完整） |

### 低敏感级别（不应含敏感信息）

| 目录 | 说明 |
|------|------|
| `logs\` | 操作日志，不应记录完整敏感值 |
| `api_logs\` | API 操作日志，不应记录完整敏感值 |

---

## 二、Markdown 报告 vs JSON 报告

工具生成的报告有两种格式：

| 格式 | 敏感信息处理 | 可以给谁看 |
|------|-------------|-----------|
| `.md`（Markdown） | 已脱敏（身份证号 → `110***********1234`） | 可以放心查看 |
| `.json`（JSON） | **完整未脱敏** | 只能自己看 |

> ⚠️ 不要把 `.json` 报告发给别人或上传到公开平台。

---

## 三、不要把这些目录发给别人

| 目录 | 原因 |
|------|------|
| `vault\` | 你的完整个人资料库 |
| `input_forms\` | 包含你以前填过的真实表格 |
| `new_forms\` | 可能包含部分填写的信息 |
| `candidate_reviews\` | JSON 文件中含完整敏感值 |
| `final_outputs\` | 填好的最终表格 |
| `api_previews\` | 历史预览目录，可能含脱敏不完全的信息（可由 Cleaner 清理） |

---

## 四、备份建议

### 可以安全备份的

- 整个 `D:\SafeFill\` 目录（在本地加密存储的前提下）
- 建议使用 Windows 自带的文件历史记录或 BitLocker 加密

### 备份时注意

- 如果备份到移动硬盘或 U 盘，确保设备本身是安全的
- 如果备份到云存储（OneDrive 等），注意 `vault\` 中的文件包含完整敏感信息
- 建议在备份前确认哪些 JSON 文件包含敏感信息

---

## 五、删除资料库

如果你需要删除本地资料库：

1. 先确认是否需要备份（见上一节）
2. 删除 `vault\profiles\person_*.json`
3. 不要删除 `vault\profiles\profile_template.json`（这是模板，不含个人数据）
4. 如果想彻底清除所有痕迹，还需清理 `candidate_reviews\` 中的 JSON 文件

---

## 六、API 使用前的安全检查清单

如果你打算使用 API 辅助功能，发送前请检查：

- [ ] 已确认 `api_config.json` 中 `user_accepts_api_data_risk=true`
- [ ] API Key 已通过环境变量设置（未写入配置文件）
- [ ] 了解 API-first 模式下资料会发送给配置的大模型服务商
- [ ] 已检查 `api_logs\` 中的调用记录

---

## 七、如果怀疑数据泄露

1. 检查 `logs\` 和 `api_logs\` 目录中的日志文件
2. 检查是否有异常的联网记录（日志中会记录）
3. 检查 `api_results\` 中的 API 返回结果
4. 确认 `api_config.json` 中 `enabled` 是否为 `false`
5. 如果 API 曾开启，检查 `api_logs\` 和 `api_results\` 了解发送了什么
