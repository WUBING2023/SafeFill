# SafeFill 主动沟通规则

SafeFill 的用户不一定懂代码，也不应该从终端日志里猜流程状态。每个子 Skill 都要主动说明当前步骤、结果、风险和下一步。

## 每次开始前

必须告诉用户：

1. 当前调用哪个子 Skill。
2. 这一步解决什么问题。
3. 会读取哪些目录。
4. 会修改哪些目录，哪些目录不会修改。
5. 是否联网，是否调用 API。

## 每次运行后

必须告诉用户：

1. 本步是否成功。
2. 生成了哪些关键文件，以及这些文件干什么用。
3. 是否发现风险、冲突、缺失信息。
4. 用户现在需要做什么。
5. 建议下一步调用哪个 SafeFill 子 Skill。

## 需要用户亲自确认的地方

以下操作不能由 Claude Code 代替用户确认：

1. 保存或覆盖 vault 中的个人资料。
2. 在网页中点击“保存检查结果”。
3. 允许 API send。
4. 执行 Cleaner clean 删除临时文件。

## 标准话术

### 开始 SafeFill

```
欢迎使用 SafeFill。我会按“提取资料 → 保存资料库 → 填表检查 → 最终导出”的顺序帮你处理。
API-first 模式。不修改原始文件。个人资料只保存在本地 vault。API数据风险由用户承担。
我会先检查当前项目状态，然后主动建议下一步。
```

### ProfileExtract 后

```
我已经从旧资料中提取到候选信息。这些信息还没有写入 vault。
请重点检查姓名、手机号、身份证号、邮箱、单位、专业等字段是否正确。
建议下一步：运行 SafeFill-ProfileSave，把确认后的信息保存到 vault\profile.json。
```

### ProfileSave 后

```
个人信息库已经更新/保持不变。现在 vault 中只有一个正式个人信息表：vault\profile.json。
建议下一步：把需要填写的空白表格放入 new_forms，然后运行 SafeFill-FormReview。
```

### FormReview 后

```
我已经生成网页检查界面。请在浏览器中检查表格内容，可以直接修改黄色空白格。
检查完成后，请点击“保存检查结果”。只有点击保存后，FinalExport 才会读取你的确认内容。
建议下一步：保存网页检查结果后运行 SafeFill-FinalExport。
```

### FinalExport 后

```
最终文件已经生成在 final_outputs。
建议你先打开最终 Word/Excel 检查格式和内容。
如果确认本轮完成，可以运行 SafeFill-Cleaner preview 查看可清理的临时隐私文件。
```

## 禁止

- 不得只输出脚本日志。
- 不得只说“完成”。
- 不得让用户自己猜下一步。
- 不得绕过用户点击保存、确认覆盖、确认清理等动作。
