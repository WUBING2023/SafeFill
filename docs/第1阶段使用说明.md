# 第 1 阶段使用说明 — 本地资料库初始化

> 本文档面向不会代码的用户。请按步骤操作，不需要任何编程知识。

---

## 本阶段做什么？

把你以前填过的旧表格（Word 或 Excel）放到工具目录里，工具会自动提取里面可能是你个人信息的字段，生成一份检查清单给你核对。你确认无误后，工具把这些信息保存到你自己的本地资料库中。

**本阶段不会自动填新表。** 自动填表功能在后续阶段实现。

---

## 第 1 步：准备旧表格文件

把你之前填过的表格文件复制到以下目录：

```
D:\SafeFill\input_forms\
```

### 支持的文件格式

| 格式 | 说明 | 示例 |
|------|------|------|
| `.docx` | Word 文档 | 项目申报表.docx、个人信息表.docx |
| `.xlsx` | Excel 表格 | 人员信息登记表.xlsx、成果汇总表.xlsx |

### 不支持的文件格式

- ❌ `.pdf` — 本阶段不支持，后续版本会支持
- ❌ 扫描件（如扫描的 PDF 或图片）— 本阶段不支持
- ❌ 图片表格（如手机拍的表格照片）— 本阶段不支持
- ❌ 旧版 `.doc` 文件 — 请先在 Word 中另存为 `.docx`

### ⚠️ 重要提醒

- 放入 `input_forms` 的是**副本**，工具不会修改你的原始文件。
- 如果不需要提取，就不要放文件进去。
- 空的 `input_forms` 目录是正常的，工具会提示你放入文件。

---

## 第 2 步：运行候选信息提取

### 如何运行

1. 打开 Windows 命令提示符（在开始菜单搜索 `cmd` 或 `PowerShell`）。
2. 输入以下命令并回车：

```
python D:\SafeFill\app\extract_candidates.py
```

### 运行后会发生什么

1. 工具会扫描 `input_forms` 中的所有 `.docx` 和 `.xlsx` 文件。
2. 工具会查找可能是你个人信息的字段（如姓名、手机号、工作单位等）。
3. 工具会在 `candidate_reviews` 目录中生成两个文件：
   - `candidate_review_时间戳.md` — **检查文件**（敏感信息已脱敏，可放心查看）
   - `candidate_review_时间戳.json` — **数据文件**（含完整信息，仅本地保存）

### 如果提示"没有发现可处理文件"

说明 `input_forms` 目录是空的，或者里面的文件不是 `.docx` / `.xlsx` 格式。把旧表格放进去后重新运行即可。

---

## 第 3 步：检查候选信息

### 打开检查文件

进入文件夹：
```
D:\SafeFill\candidate_reviews\
```

双击打开 `candidate_review_时间戳.md` 文件（可用记事本或任何文本编辑器打开）。

### 检查什么

表格列出了工具提取的每个字段：

| 列 | 说明 |
|------|------|
| 字段 | 字段的中文名称 |
| 候选值 | 工具提取到的内容（敏感信息已脱敏，如 `138****5678`） |
| 来源文件 | 这个值是从哪个表格文件提取的 |
| 是否敏感 | 标记 🔴 的是敏感信息 |
| 置信度 | "高"表示比较准确，"需确认"表示建议仔细核对 |
| 是否需要确认 | 标记 ⚠️ 的字段建议逐项核实 |

### 常见问题

- **某个字段没提取到**：工具可能无法识别你表格中的字段名格式。可以在下一步手动补充。
- **候选值不准确**：工具按字段名匹配提取的，如果表格里有多个相似字段，可能提取到不正确的值。请在下一步修正。
- **项目经历/个人简介标记为"需确认"**：这些字段内容较长，工具给的置信度较低，请仔细核对。

---

## 第 4 步：确认并整理信息

### 创建确认文件

在 `D:\SafeFill\candidate_reviews\` 目录中，创建一个名为 **`confirmed_profile.json`** 的文件。

你可以：
- **方法一**：复制 `candidate_review_时间戳.json`，重命名为 `confirmed_profile.json`，然后编辑。
- **方法二**：新建一个文本文件，按下面的格式编写，保存为 `confirmed_profile.json`。

### confirmed_profile.json 格式

```json
{
  "candidates": {
    "name":                 { "value": "张老师", "confirmed": true },
    "gender":               { "value": "男", "confirmed": true },
    "birth_date":           { "value": "1980-01", "confirmed": true },
    "id_number":            { "value": "你的身份证号", "confirmed": true },
    "phone":                { "value": "你的手机号", "confirmed": true },
    "email":                { "value": "zhang@example.com", "confirmed": true },
    "organization":         { "value": "某某大学", "confirmed": true },
    "department":           { "value": "某某学院", "confirmed": true },
    "title":                { "value": "教授", "confirmed": true },
    "education":            { "value": "博士", "confirmed": false },
    "degree":               { "value": "博士", "confirmed": false },
    "research_area":        { "value": "计算机科学", "confirmed": true },
    "address":              { "value": "某市某区某路某号", "confirmed": false },
    "photo_path":           { "value": "", "confirmed": false },
    "project_experience":   { "value": "你的项目经历", "confirmed": false },
    "biography":            { "value": "你的个人简介", "confirmed": false }
  }
}
```

### 关键规则

1. **`"confirmed": true`** — 只有设为 `true` 的字段才会被保存到资料库。
2. **`"confirmed": false`** — 设为 `false` 的字段不会被保存，即使有值。
3. **空值不会被保存** — 如果 `"value"` 为空，即使 `"confirmed": true` 也不会保存。
4. **不确定的字段不要确认** — 拿不准的字段请设为 `false`，以后可以再补充。

---

## 第 5 步：保存到资料库

### 运行保存脚本

在命令提示符中输入：

```
python D:\SafeFill\app\save_confirmed_profile.py
```

### 运行后会发生什么

1. 工具读取 `confirmed_profile.json`。
2. 检查每个字段是否已确认。
3. 跳过空字段和未确认字段。
4. 将确认的字段保存到：
   ```
   D:\SafeFill\vault\profiles\person_001.json
   ```
5. 下次保存时，自动生成 `person_002.json`、`person_003.json`…，不会覆盖已有资料。

### 如果提示错误

| 错误提示 | 解决方法 |
|----------|----------|
| "confirmed_profile.json 不存在" | 请先完成第 4 步，创建该文件 |
| "不是有效的 JSON 文件" | 检查文件格式，确保括号和引号正确 |
| "没有可保存的字段" | 检查是否有字段设为 `"confirmed": true` 且值不为空 |

## 目录说明（参考）

| 目录 | 用途 |
|------|------|
| `input_forms\` | **放旧表格**的地方（只放副本，不修改原始文件） |
| `candidate_reviews\` | 工具生成的**候选信息检查文件** |
| `vault\profiles\` | **你的资料库**（只保存你确认过的信息） |
| `vault\photos\` | 证件照存放（第一版仅记录路径） |
| `logs\` | 操作日志（不记录身份证号等敏感内容） |
| `app\` | 程序代码（不需要动） |
| `docs\` | 说明文档 |

---

## 安全说明

本阶段**完全离线运行**，不会发生以下任何行为：

- ❌ 不会联网（不需要网络连接即可使用）
- ❌ 不会修改你的原始文件
- ❌ 不会删除你的任何文件
- ❌ 不会自动保存未经你确认的信息
- ❌ 不会把数据发送到任何服务器

你确认保存的资料**只存储在你自己电脑的 `vault\` 目录中**，不会离开你的电脑。
