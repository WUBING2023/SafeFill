# 安全设计

## 1. 安全目标

本项目的核心难点不是填表逻辑，而是**让用户相信资料不会被泄露**。

目标用户（老师、科研人员、行政人员）处理的表格中往往包含高度敏感的个人信息（身份证号、银行账号、未公开项目材料等），一旦泄露可能导致身份盗用、学术成果被窃取等严重后果。因此，安全设计必须是本项目的第一优先级，贯穿从设计到实现的每一个环节。

### 安全设计原则

- **默认拒绝（Deny by Default）**：任何联网行为默认禁止，仅当用户显式授权后才允许。
- **最小权限（Least Privilege）**：工具仅访问完成填表任务所必需的最少文件和数据。
- **用户主导（User-in-the-Loop）**：所有敏感操作（保存、发送、导出）必须由用户逐项确认。
- **API-first**：主流程默认使用大模型 API 处理，用户自行承担 API 数据风险。本地规则作为 fallback。

## 2. 本地隔离目录设计

### 目录结构

```
D:\SafeFill\
├── SKILL.md                  # Skill 说明文档（本文件）
├── docs\                     # 设计文档
│   ├── 需求说明.md
│   ├── 安全设计.md
│   ├── 数据字段清单.md
│   └── 第0阶段验收清单.md
├── app\                      # 程序代码（后续阶段）
├── vault\                    # 用户资料库（敏感数据）
│   ├── profiles\             # 用户确认过的个人信息档案
│   └── photos\               # 用户确认过的证件照
├── input_forms\              # 导入的原始表格副本
├── draft_outputs\            # 自动填写草稿
├── final_outputs\            # 用户确认后的最终文件
└── logs\                     # 本地操作记录
```

### 各目录用途说明

| 目录 | 用途 | 安全级别 | 说明 |
|------|------|----------|------|
| `vault/` | 保存用户确认过的资料 | **高** | 包含用户个人信息，应限制访问权限 |
| `vault/profiles/` | 个人信息档案文件 | **高** | 每个文件对应一组用户确认过的信息 |
| `vault/photos/` | 证件照文件 | **高** | 仅保存路径，不处理图片内容（第一版） |
| `input_forms/` | 导入的原始表格副本 | **中** | 原始文件的只读副本，永不修改 |
| `draft_outputs/` | 自动填写草稿 | **中** | 机器生成的填写草稿，待用户审核 |
| `final_outputs/` | 用户确认后的最终文件 | **中** | 用户确认无误后的导出文件 |
| `logs/` | 本地操作记录 | **中** | 仅记录操作日志，不记录敏感字段内容 |
| `docs/` | 设计文档 | **低** | 不含任何真实个人信息 |
| `app/` | 程序代码（后续阶段） | **低** | 不含用户数据 |

### 目录安全规则

1. `vault/` 目录不得被自动同步到云存储（如 OneDrive、iCloud 等）。
2. 程序不得读取 `vault/` 之外的私人文件。
3. `logs/` 中的日志不得记录身份证号、手机号、住址等敏感字段的完整内容。
4. `draft_outputs/` 和 `final_outputs/` 中的文件不得自动上传到任何远程服务。

## 3. 联网规则

### 3.1 默认状态：完全离线

工具安装后默认处于**完全离线模式**，不发起任何网络请求。所有填表功能（资料导入、字段匹配、草稿生成、导出）均在本地完成。

### 3.2 API 作为唯一联网接口

当且仅当用户需要以下功能时，可通过 API 接口联网：

- 调用大语言模型辅助理解非标准表格结构（第二版及以后）
- 调用 OCR 服务识别扫描件文字（第三版及以后）

### 3.3 API 调用规则

1. **API-first 模式**：主流程默认使用大模型 API。`user_accepts_api_data_risk` 设为 true 后生效。
2. **用户手动开启**：如需使用 API，用户必须手动在设置中开启，并提供 API 端点地址和密钥。
3. **发送前展示内容**：每次 API 调用前，工具必须以弹窗或页面形式展示完整的待发送数据，不得隐藏任何内容。
4. **用户逐项确认**：用户必须逐项检查待发送内容，确认无误后点击"发送"按钮，工具才能发起请求。
5. **敏感字段自动脱敏**：工具必须内置敏感字段检测逻辑，以下字段在发送给 API 前自动脱敏或拦截：
   - 身份证号 → 拦截，不允许发送
   - 手机号 → 拦截，不允许发送
   - 家庭住址 / 通讯地址 → 拦截，不允许发送
   - 证件照 → 拦截，不允许发送
   - 银行账号 → 拦截，不允许发送
   - 签名图片 → 拦截，不允许发送
6. **禁止上传完整原始表格**：API 调用仅允许发送需要匹配的字段名称和类型，不得将完整原始表格内容作为上下文发送。
7. **禁止后台静默联网**：工具不得在后台静默发起网络请求。任何联网操作必须在 UI 上有明确可见的进度指示。

### 3.4 联网白名单（后续版本）

若后续版本支持 API 功能，联网请求仅允许访问：

- 用户在设置中指定的 API 端点
- 用户显式导入的 OCR 服务地址

其他所有域名和 IP 地址的请求一律拦截。

## 4. 敏感信息规则

### 4.1 敏感信息清单

以下信息被定义为本项目中的**敏感信息**，受到最高级别的保护：

| 敏感信息类型 | 保护级别 | 处理规则 |
|-------------|----------|----------|
| 身份证号 | **最高** | 禁止发送给 API；日志中仅记录是否已填写，不记录内容 |
| 手机号 | **最高** | 禁止发送给 API；日志中仅记录是否已填写，不记录内容 |
| 家庭住址 / 通讯地址 | **最高** | 禁止发送给 API；日志中不记录 |
| 证件照 | **最高** | 禁止发送给 API；第一版仅保存路径 |
| 银行账号 | **最高** | 禁止发送给 API；日志中不记录 |
| 签名图片 | **最高** | 禁止发送给 API |
| 完整个人简历 | **高** | 仅可保存在本地 vault，禁止发送给 API |
| 未公开项目申报材料 | **高** | 仅可保存在本地 input_forms，禁止发送给 API |

### 4.2 敏感信息处理规则

1. **存储**：敏感信息仅存储在 `vault/` 目录中，文件权限应设为仅当前用户可读。
2. **传输**：敏感信息不得通过 API 发送给任何远程服务。
3. **日志**：操作日志中不得记录敏感信息的具体内容，仅记录操作类型和时间。
4. **导出**：导出到 `final_outputs/` 的文件如果包含敏感信息，工具应在导出前提示用户注意文件安全。
5. **清理**：用户应能随时删除 `vault/` 中的任意资料，删除操作应不可恢复。

## 5. 原始文件保护规则

### 5.1 文件保护原则

1. **永远不覆盖用户上传的原始文件**：原始文件导入后，所有处理操作在副本上进行。
2. **导入时自动创建副本**：原始文件导入到 `input_forms/` 时即创建只读副本，原始文件路径记录在日志中但不被修改。
3. **草稿输出到 draft_outputs**：所有自动填写生成的草稿保存到 `draft_outputs/`，不与原始文件混放。
4. **最终文件输出到 final_outputs**：用户确认后的最终文件保存到 `final_outputs/`。
5. **修改前必须生成副本**：任何对表格内容的修改操作，必须在副本上执行。
6. **导出失败不能破坏原始文件**：如果导出过程中发生错误，仅清理失败的临时文件，不得触及原始文件。

### 5.2 文件操作流程图

```
原始 .docx / .xlsx
    │
    ├──→ 只读读取表格内容
    │
    └──→ 复制到 input_forms/（只读副本）
              │
              └──→ 提取字段 → 匹配资料库 → 生成草稿
                                                │
                                                └──→ draft_outputs/（填写草稿）
                                                          │
                                                          └──→ 用户检查确认
                                                                    │
                                                                    └──→ final_outputs/（最终文件）
```

## 6. 用户确认规则

### 6.1 需要用户确认的操作

| 操作 | 确认方式 | 默认行为 |
|------|----------|----------|
| 初始化资料库 | 逐项确认哪些字段保存 | 全部不保存 |
| 从旧表格导入信息 | 逐项确认哪些提取结果保存 | 全部不保存 |
| 自动填写草稿 | 本地网页逐项检查 | — |
| 补充未知信息 | 逐项填写 | 默认不保存到资料库 |
| 保存补充信息到资料库 | 逐项勾选 | 不勾选（不保存） |
| API 调用（后续版本） | 展示待发送内容，逐项确认 | 不发送 |
| 导出最终文件 | 确认导出路径和文件名 | — |
| 删除资料库信息 | 二次确认 | 取消 |

### 6.2 关键确认规则

1. **初始化资料库时**：用户必须逐项确认哪些信息可以保存到 `vault/`。没有用户的明确勾选，任何信息不得写入资料库。
2. **自动填表后**：生成的草稿必须在本地网页中展示，用户逐项核对后方可导出。工具不得跳过检查步骤直接导出。
3. **未知字段**：工具无法匹配的字段必须在检查页面中高亮显示，让用户手动补充。
4. **补充信息的保存**：用户补充的信息**默认不保存**到资料库。仅当用户明确勾选"保存到资料库"复选框后，该字段才会被写入 `vault/`。
5. **API 调用前（后续版本）**：工具必须展示完整的待发送数据，用户逐项确认后才能发送。不得提供"一键发送"或"不再提示"等跳过确认的选项。
