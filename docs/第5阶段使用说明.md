# 第 5 阶段使用说明 — API 辅助理解字段

> 本文档面向不会代码的用户。请按步骤操作。
> **API 不是必须使用的功能。** 绝大多数表格用本地规则就能处理。只有遇到无法理解的字段名时才考虑使用。

---

## 本阶段做什么？

当表格中的字段名比较特殊（例如非标准命名），本地规则无法判断它对应你的哪个资料库字段时，API 辅助可以帮你：
1. 生成"准备发给 AI 分析"的预览内容
2. 你确认内容安全后，调用 AI 帮忙判断字段含义
3. AI 返回建议，你参考建议手动调整

**AI 只是给建议，不会自动帮你改文件。**

---

## 前提条件

1. 已启用 API 功能（默认为关闭状态）。
2. 已从模板创建配置文件。
3. 已设置 API Key 环境变量。

---

## 第 1 步：创建配置文件

从模板复制配置文件：

```
复制：D:\SafeFill\app\api_config_template.json
粘贴为：D:\SafeFill\app\api_config.json
```

用记事本打开 `api_config.json`，根据需要编辑：

```json
{
  "enabled": false,
  "provider": "openai_compatible",
  "endpoint": "",
  "model": "",
  "api_key_env": "SECURE_FORM_API_KEY",
  "allow_sensitive": false,
  "max_chars_per_request": 3000,
  "dry_run": true
}
```

> **重要**：不要在 `api_config.json` 中填写真实 API Key！

---

## 第 2 步：设置 API Key（环境变量）

API Key 只能通过 Windows 环境变量设置。

### 临时设置（仅当前命令提示符窗口有效）

```
set SECURE_FORM_API_KEY=你的密钥
```

### 永久设置

1. 搜索"编辑系统环境变量"。
2. 点击"环境变量"。
3. 新建用户变量：变量名 `SECURE_FORM_API_KEY`，值填你的密钥。

---

## 第 3 步：开启 API 功能

编辑 `api_config.json`：
- 把 `"enabled"` 改为 `true`
- 填写 `endpoint`（API 地址，例如 `https://api.openai.com/v1/chat/completions`）
- 填写 `model`（模型名称，例如 `gpt-4o`）
- `"dry_run"` 先保持 `true`（安全模式，不真正联网）

---

## 第 4 步：生成 API 预览

运行：
```
python D:\SafeFill\app\api_assist.py preview
```

工具会：
1. 读取最新的填写报告。
2. 找出本地规则无法匹配的字段。
3. 自动脱敏敏感信息。
4. 生成预览文件到 `api_previews\`。

预览文件有两个：
- `api_preview_时间戳.md` — 人类可读，**建议优先看这个**
- `api_preview_时间戳.json` — 机器可读

---

## 第 5 步：检查预览内容

打开 `api_previews\` 中的 Markdown 文件，检查：

1. 待发送的字段名是否正确。
2. 有没有不希望发送的信息。
3. 敏感字段是否已拦截（安全闸门部分会标注）。
4. 信息总长度是否合理。

---

## 第 6 步：确认并真正发送

**仅当你确认预览内容安全、且确实需要 AI 辅助时，才继续。**

### 6.1 确认预览

打开对应的 JSON 预览文件，找到：
```json
"approved_by_user": false
```
改为：
```json
"approved_by_user": true
```

### 6.2 关闭干运行模式

编辑 `api_config.json`：
```json
"dry_run": false
```

### 6.3 发送请求

```
python D:\SafeFill\app\api_assist.py send D:\SafeFill\api_previews\api_preview_xxx.json
```

**必须通过全部 6 道安全闸门才会发送：**
1. enabled = true
2. dry_run = false
3. approved_by_user = true
4. 无未拦截敏感字段
5. API Key 已设置
6. endpoint 已填写

---

## 第 7 步：查看 AI 结果

结果保存在 `api_results\`：
- `api_result_时间戳.json` — 结构化结果
- `api_result_时间戳.md` — 可读报告

结果中包含每个字段的匹配建议、置信度、理由。

### 重要

- AI 的建议**仅供参考**，不自动写入表格。
- 你仍需在第 3 阶段网页中人工确认。
- 置信度低于 0.5 的建议建议忽略。

---

## 安全提醒

| 项目 | 说明 |
|------|------|
| API 默认关闭 | 不手动开启就不会联网 |
| 预览先于发送 | 每次发送前必须先生成预览 |
| 用户确认必须 | 不手动改 `approved_by_user` 就发不出去 |
| 敏感信息不发送 | 身份证号、手机号、地址等被自动拦截 |
| API Key 不入文件 | 只在环境变量中 |
| 结果不自动生效 | AI 建议需要你人工判断 |

---

## 常见问题

| 问题 | 解决方法 |
|------|----------|
| "配置文件不存在" | 从 api_config_template.json 复制创建 api_config.json |
| "[STOP] enabled=false" | 编辑 api_config.json，设 enabled 为 true |
| "[STOP] dry_run=true" | 编辑 api_config.json，设 dry_run 为 false |
| "[STOP] approved_by_user=false" | 编辑预览 JSON，设 approved_by_user 为 true |
| "[STOP] API Key 未设置" | 设置环境变量 SECURE_FORM_API_KEY |
| "API 请求失败" | 检查 endpoint 和 API Key 是否正确，确认网络连通 |
| "不需要 API" | 完全正常，主流程不依赖 API |
