# 第 5 阶段 — API 安全设计

## 1. 设计原则

**API 不是核心功能，是可选的辅助工具。** 整个工具在无 API 的情况下已经可以完成完整的填表闭环（第 1→4 阶段）。

API 的唯一用途：当本地规则无法判断表格字段含义时，辅助用户理解"这个字段可能对应资料库哪个字段"。

---

## 2. 安全闸门机制

API 请求在发送前必须依次通过 **六道安全闸门**，任意一道未通过即拒绝发送：

### 闸门 1：enabled = true

- 默认值：`false`
- 用户必须手动编辑 `api_config.json`，将 `enabled` 改为 `true`。
- 即使通过此闸门，也不代表一定会联网（见闸门 2）。

### 闸门 2：dry_run = false

- 默认值：`true`
- 干运行模式下，`preview` 命令只生成本地预览文件，不发起任何网络请求。
- 用户必须将 `dry_run` 改为 `false` 才能通过此闸门。

### 闸门 3：approved_by_user = true

- 预览 JSON 文件中此字段默认为 `false`。
- 用户必须先运行 `preview` 生成预览文件，打开 Markdown 检查待发送内容。
- 确认内容安全后，用户必须手动将预览 JSON 中的 `approved_by_user` 改为 `true`。

### 闸门 4：无敏感字段

- 敏感字段清单：身份证号、手机号、通讯地址、证件照、银行账号、签名图片。
- 预览生成时，工具自动检测并脱敏/拦截敏感字段。
- 如果 `allow_sensitive` 为 `false`（默认），包含敏感字段的请求将被阻止。

### 闸门 5：API Key 已设置

- API Key 只能通过环境变量 `SECURE_FORM_API_KEY` 设置。
- 工具不提供任何在文件中存储 API Key 的途径。
- 配置文件只记录环境变量名，不记录 Key 值。

### 闸门 6：endpoint 已设置

- `api_config.json` 中的 `endpoint` 不能为空。
- 防止误配置导致请求发送到错误的地址。

---

## 3. 敏感信息不发送规则

以下内容在任何情况下都**禁止**发送给 API：

| 数据类型 | 处理方式 | 拦截级别 |
|----------|----------|----------|
| 身份证号（完整） | 拦截，generation 中用 `[敏感字段已脱敏]` 替代 | **最高** |
| 手机号（完整） | 拦截 | **最高** |
| 通讯地址（完整） | 屏蔽，仅保留前 6 字 + `***` | **最高** |
| 证件照路径 | 屏蔽，仅提示"存在" | **最高** |
| 证件照图片内容 | 永不发送 | **最高** |
| 银行账号 | 拦截 | **最高** |
| 完整个人简历 | 严格限制字符数，默认不发送 | **高** |
| 完整原始表格 | 禁止发送，仅发送字段名称列表 | **最高** |

---

## 4. API 只能做什么

1. 接收表格字段名列表和脱敏后的候选值。
2. 推荐字段与资料库字段的对应关系。
3. 返回结构化建议（含置信度和理由）。
4. 对于不确定的匹配返回 `requires_user_review: true`。

## 5. API 禁止做什么

1. 自动填写表格文件。
2. 自动修改任何本地文件。
3. 自动保存信息到 vault。
4. 自动提交申报系统。
5. 主动搜索用户个人信息。
6. 接收完整敏感资料。
7. 接收完整原始表格内容。
8. 后台静默发送请求（必须有终端输出和日志记录）。

---

## 6. API Key 保护

- API Key **只能**通过环境变量设置：`SECURE_FORM_API_KEY`
- 配置文件 `api_config.json` 只存储环境变量名，不存储 Key 值。
- `api_config_template.json` 中不包含任何真实 Key。
- 日志中不记录 API Key。
- Markdown 报告中不包含 API Key。

---

## 7. 日志与审计

每次 API 操作（preview 和 send）必须写入 `api_logs\`：

| 记录内容 | 记录方式 |
|----------|----------|
| 操作时间 | 完整记录 |
| 操作类型（preview/send） | 完整记录 |
| 是否联网 | 完整记录 |
| 使用的预览文件 | 路径记录 |
| 输出结果文件 | 路径记录 |
| 拦截的敏感字段数 | 计数，不记录值 |
| 阻止原因 | 记录闸门名称 |
| 身份证号 | **不记录** |
| 手机号 | **不记录** |
| 完整地址 | **不记录** |
| API Key | **不记录** |
| 证件照路径 | **不记录** |

---

## 8. 与后续阶段的关系

第 5 阶段仅提供 **字段理解辅助**。即使 API 返回了建议，用户也必须在第 3 阶段网页中人工确认后，才能通过第 4 阶段导出最终文件。
