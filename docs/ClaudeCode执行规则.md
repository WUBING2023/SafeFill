# Claude Code 执行规则

> 本文件规定 Claude Code 在操作"SafeFill"项目时必须遵守的行为边界。任何人将本 Skill 交给 Claude Code 执行时，都应先让 Claude Code 读取本文件。

---

## 一、开始前必读

Claude Code 在执行本 Skill 的任何操作前，必须先读取：

```
D:\SafeFill\docs\项目阶段总路线图.md
```

该文件记录了项目总进度。如果因上下文压缩或会话恢复导致记忆丢失，读此文件即可了解当前状态。

---

## 二、逐阶段执行

| 规则 | 说明 |
|------|------|
| 不得越阶段 | 一次只执行一个阶段。用户说"做第 3 阶段"，就只做第 3 阶段相关的操作。 |
| 不得合并阶段 | 不要因为"看起来简单"就把两个阶段合在一起做。 |
| 不得跳阶段 | 各阶段有前提条件（如前阶段产物），跳过可能导致数据丢失。 |
| 阶段完成必须汇报 | 报告创建/修改/联网/API/依赖/vault/原始文件状态。 |

---

## 三、文件操作约束

| 禁止 | 说明 |
|------|------|
| 修改核心脚本 | `app\*.py` 不得随意改动，除非用户明确要求且只改指定位置 |
| 覆盖已有文件 | 修改前先备份到 `docs\backups\` |
| 删除用户数据 | `vault\`、`input_forms\`、`new_forms\` 等目录永远不动 |
| 读取项目外文件 | 仅限 `D:\SafeFill\` 内 |
| 把用户数据打包 | 不得把 vault/input_forms/new_forms 等打包或复制到项目外 |

---

## 四、网络安全约束

| 禁止 | 说明 |
|------|------|
| 默认联网 | 所有操作本地完成 |
| 自动启用 API | API 需手动配置才可启用 |
| 跳过 API 预览 | 必须先 `preview`，用户确认后才能 `send` |
| 后台静默联网 | 任何联网操作必须有终端输出和日志记录 |
| 保存 API Key | API Key 只能通过环境变量 `SECURE_FORM_API_KEY` 设置，不得写入任何文件 |

---

## 五、依赖管理约束

| 规则 | 说明 |
|------|------|
| 不得安装依赖 | 除非用户明确同意安装指定包 |
| 先检查再报告 | `import` 失败时报告缺少什么，不自行 `pip install` |
| 只用标准库 | 第 3 阶段网页服务器只用 `http.server`，不得引入 Flask 等 |

---

## 六、隐私数据约束

| 规则 | 说明 |
|------|------|
| 不完整展示敏感信息 | 发现身份证号、手机号等，只报告字段类型和位置，不输出值 |
| 日志不记录敏感值 | 日志只记录操作类型，不记录身份证号/手机号/地址 |
| 不生成真实隐私数据 | 测试用数据必须明确标注为"测试用" |
| Markdown 报告脱敏 | 对外展示的报告中敏感信息必须脱敏 |
| 发现泄露风险停止 | 如发现敏感信息可能泄露的路径，立即停止并报告 |

---

## 七、处理真实隐私数据时

如果 Claude Code 在处理过程中发现用户真实数据（如 `vault\person_001.json` 中的身份证号）：

1. 不要完整输出到终端或日志。
2. 报告格式：`发现敏感字段 [字段名]，位于 [文件路径]，值已脱敏处理。`
3. 不要把这些数据作为上下文示例发给外部服务。
4. 如果需要展示，只展示脱敏后的版本。

---

```
- 是否修改原始文件：（是/否）
- 是否修改草稿文件：（是/否）
- 风险提示：（如有）
```
