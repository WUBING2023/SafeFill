# 第 3 阶段使用说明 — 本地网页检查和补充

> 本文档面向不会代码的用户。请按步骤操作，不需要任何编程知识。

---

## 本阶段做什么？

你已经完成了第 2 阶段，`filling_reports\` 目录中有自动填写报告。本阶段的任务是：**启动一个只在本机运行的网页，让你在浏览器里逐项检查自动填写的内容，修改错误、补充遗漏，并选择哪些可以保存到资料库。**

本阶段**只做检查确认**，不修改原始文件、不修改草稿、不输出最终文件。

---

## 前提条件

1. 已完成第 2 阶段。
2. `D:\SafeFill\filling_reports\` 目录中已有 `.json` 填写报告。
3. 如果还没有报告，请先运行：
   ```
   python D:\SafeFill\app\fill_form_draft.py
   ```

---

## 第 1 步：启动本地网页服务

打开命令提示符（`cmd` 或 `PowerShell`），输入：

```
python D:\SafeFill\app\review_server.py
```

启动后终端会显示：

```
=======================================================
  本地安全填表检查服务
  =============================================
  地址: http://127.0.0.1:8787/
  绑定: 127.0.0.1（仅本机可访问）
  联网: 否
  API:  否
  =============================================
```

### 如果端口被占用

如果 8787 端口被其他程序占用，工具会自动尝试 8788、8789。
终端会提示实际使用的端口号，请按提示访问对应地址。

---

## 第 2 步：打开浏览器

在浏览器地址栏输入：

```
http://127.0.0.1:8787/
```

> ⚠️ 必须是 `127.0.0.1`，不能用 `localhost`（部分系统可能解析到 IPv6）。

### 页面布局

| 区域 | 位置 | 内容 |
|------|------|------|
| 顶部状态栏 | 最上方 | 安全状态标识（仅本机运行、API-first、用户确认发送） |
| 报告列表 | 左侧 | 所有填写报告，点击选择 |
| 字段列表 | 中间 | 已填写 / 需复核 / 未填写 / 已有内容未覆盖 |
| 字段详情 | 右侧 | 点击某个字段后，显示详情和编辑区 |
| 底部操作栏 | 最下方 | 当前统计 + 保存检查结果按钮 |

---

## 第 3 步：选择报告

在左侧报告列表中，点击一个报告。

页面会展示四类字段：

### ✅ 已填写字段
系统自动填写的内容。你需要**逐项检查是否准确**。

对每个字段可以：
- **接受**：确认无误，加入确认列表。
- **修改**：输入新值替换当前值。
- **清空**：把内容清掉。
- **拒绝**：标记为不填。

### ⚠️ 需复核字段
长文本字段（个人简介、项目经历等），系统已填写但建议仔细核对。
操作方式同"已填写字段"。

### ❌ 未填写字段
系统识别到表格中有这个字段名，但资料库中没有对应信息。
你需要**手动补充**内容。

操作方式：
1. 点击字段
2. 在右侧输入框中填写内容
3. 点击"补充此字段"

### 🔒 已有内容未覆盖
表格中这个位置本来就有内容，系统保留了原内容、没有覆盖。
你可以选择保持原样，或者手动修改。

---

## 第 4 步：逐项检查

### 操作流程

1. 在中间区域点击一个字段，右侧会显示详情。
2. 查看当前值是否正确。
3. 如果正确 → 点击 **✅ 接受当前值**。
4. 如果有误 → 在输入框中输入新值 → 点击 **✏️ 应用修改**。
5. 如果是未填写字段 → 输入补充内容 → 点击 **✏️ 补充此字段**。
6. 如果不需要填写 → 点击 **❌ 拒绝**。
7. 如果想把补充的内容保存到资料库（以后都不用再填）→ 勾选 **"将补充/修改的内容保存到资料库"**，再点击对应操作按钮。

### ⚠️ 关于"保存到资料库"

- 勾选该选项后，这个字段**只是被记录下来**。
- 实际上**不会立即**写入 `vault\profiles\`。
- 真正写入资料库需要在后续阶段手动执行。
- 这样做是为了防止误操作覆盖你的资料库。

---

## 第 5 步：保存检查结果

完成所有需要检查的字段后，点击右下角的 **💾 保存检查结果** 按钮。

保存成功后，浏览器会显示输出文件路径：

```
D:\SafeFill\review_results\review_result_时间戳.json
```

### 保存了什么

| 内容 | 说明 |
|------|------|
| 已确认字段 | 你点击"接受"的字段 |
| 已修改字段 | 你修改后的新值（含旧值对比） |
| 用户补充字段 | 你在未填写字段中手动补充的内容 |
| 标记保存到资料库 | 你勾选了"保存到资料库"的字段 |
| 已拒绝字段 | 你标记为不填的字段 |
| 安全状态 | 联网: 否, API: 否 |

---

## 第 6 步：停止服务

检查完成后，回到命令提示符窗口，按 **Ctrl + C** 停止服务。

---

## 敏感信息保护

网页中对敏感信息做了脱敏显示：

| 字段 | 网页显示 |
|------|---------|
| 身份证号 | `440***********1234` |
| 手机号 | `138****5678` |
| 通讯地址 | 只显示前 6 个字 + `***` |
| 证件照路径 | 只显示"存在"或"不存在" |

如果需要修改这些字段，可以在输入框中重新输入完整值。

---

## 目录说明

| 目录 | 本阶段用途 |
|------|-----------|
| `app\web_review\` | 网页文件（HTML/CSS/JS），纯本地 |
| `filling_reports\` | 读取填写报告（只读） |
| `review_results\` | 保存你的检查结果 |
| `logs\` | 操作日志 |
| `vault\profiles\` | 本阶段**不修改** |

---

## 安全说明

本阶段**完全离线运行**：

- ❌ 不会联网（网页只在你自己的电脑上运行）
- ❌ 不会调用 API
- ❌ 不会修改原始文件
- ❌ 不会修改草稿文件
- ❌ 不会修改 vault 资料库
- ❌ 不会加载任何外部网页资源（CDN、字体、JS 库等，全部为 0）
- ❌ 不会把数据显示给局域网内其他人（只绑定 127.0.0.1）

---

## 常见问题

| 问题 | 解决方法 |
|------|----------|
| 页面打不开 | 确认终端没有报错，确认地址是 `http://127.0.0.1:8787/` |
| 端口被占用 | 工具会自动尝试 8788，查看终端提示的实际地址 |
| 报告列表为空 | 先运行 `fill_form_draft.py` 生成报告 |
| 某个字段显示不完整 | 网页显示的脱敏版本，完整值在第 2 阶段的 JSON 报告中 |
| 不小心关闭了浏览器 | 重新打开即可（不影响之前的操作记录） |
| 保存失败 | 检查 `review_results\` 目录是否存在，查看终端错误提示 |
