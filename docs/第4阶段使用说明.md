# 第 4 阶段使用说明 — 最终文件导出

> 本文档面向不会代码的用户。请按步骤操作，不需要任何编程知识。

---

## 本阶段做什么？

你已经完成了第 3 阶段，在网页中逐项检查了自动填写的内容，保存了检查结果。本阶段的任务是：**运行导出脚本，把检查结果应用到表格文件上，生成最终版的 Word 或 Excel 文件。**

本阶段是整个工具主流程的最后一步。完成后，你会得到一个可直接使用的填好的表格文件。

---

## 前提条件

1. 已完成第 3 阶段。
2. `D:\SafeFill\review_results\` 中已有 `review_result_*.json` 文件。
3. 第 2 阶段生成的草稿文件仍在 `D:\SafeFill\draft_outputs\` 中。

---

## 第 1 步：运行导出

打开命令提示符（`cmd` 或 `PowerShell`），输入：

```
python D:\SafeFill\app\export_final.py
```

### 运行后会发生什么

1. 工具自动找到最新的检查结果（`review_result_*.json`）。
2. 工具读取检查结果中引用的草稿文件。
3. 工具复制草稿 → 在副本上应用你的修改。
4. 输出最终文件到 `final_outputs\`。
5. 输出最终报告到 `final_reports\`。

---

## 第 2 步：查看最终文件

位置：
```
D:\SafeFill\final_outputs\
```

文件名格式：
```
原草稿文件名_最终版.docx
原草稿文件名_最终版.xlsx
```

打开最终文件，可以看到：
- 你确认的字段 → 保留或正确填写
- 你修改的字段 → 已用新值覆盖
- 你补充的字段 → 已填入空白处
- 你拒绝的字段 → 已清空

---

## 第 3 步：查看最终报告

位置：
```
D:\SafeFill\final_reports\
```

每次导出会生成两份报告：
- `final_report_时间戳.md` — 可读报告（推荐先看）
- `final_report_时间戳.json` — 完整数据

### 报告内容

| 内容 | 说明 |
|------|------|
| ✅ 成功写入 | 成功写入最终文件的字段 |
| ⚠️ 跳过 | 因各种原因未写入的字段 |
| ❌ 用户拒绝 | 你标记为不填、已清空的字段 |
| 🔍 无法定位 | 在表格中没找到对应位置、未能写入的字段 |
| 💾 建议保存到资料库 | 你勾选了"保存到资料库"的字段（仅记录，尚未实际写入 vault） |
| ⚠️ 长文本复查提醒 | 个人简介、项目经历等建议人工复查 |

### ⚠️ 重要

**如果报告中出现"无法定位字段"**，说明工具在表格中找不到该字段的位置。你需要：
1. 打开最终文件
2. 手动找到该字段
3. 手动填写

---

## 第 4 步：人工复查

最终文件仍建议用 Word 或 Excel 打开检查一遍，确认：

- 所有该填的字段都填上了
- 内容没有错位或串行
- 格式（字体、边框、排版）没有异常
- 长文本（个人简介、项目经历）内容完整

---

## 主流程回顾

至此整个工具的主流程已经闭环：

```
第 1 阶段：旧表格 → 提取候选 → 用户确认 → 资料库 (vault)
第 2 阶段：新表格 + 资料库 → 自动填写 → 草稿 (draft_outputs)
第 3 阶段：草稿报告 → 网页检查 → 修改确认 → 检查结果 (review_results)
第 4 阶段：检查结果 + 草稿 → 应用修改 → 最终文件 (final_outputs) ← 当前阶段
```

---

## 关于"保存到资料库"

在第 3 阶段网页中，你可能勾选了某些字段的"保存到资料库"选项。

**这些字段目前仅记录在最终报告中，尚未写入资料库。** 这意味着：
- 你下次填新表格时，这些补充信息**不会**自动出现
- 如果你希望永久保存这些信息，需要后续手动执行资料库更新

把 review_result 中的内容写入 vault 是一个需要谨慎处理的操作，将在后续提供。

---

## 安全说明

本阶段**完全离线运行**：

- ❌ 不会联网
- ❌ 不会调用 API
- ❌ 不会修改原始文件（你在 new_forms 中的原始表格）
- ❌ 不会修改草稿文件（draft_outputs 中的草稿保留不变）
- ❌ 不会修改 vault 资料库
- ❌ 不会自动保存补充信息到资料库

## 常见问题

| 问题 | 解决方法 |
|------|----------|
| "review_results 为空" | 先完成第 3 阶段，在网页中保存检查结果 |
| "草稿文件不存在" | 检查 draft_outputs 目录，确认草稿文件名与 review_result 中记录的一致 |
| 最终文件中某个字段没填上 | 查看 final_report 的"无法定位字段"部分，手动补填 |
| 最终文件名冲突 | 工具会自动添加时间戳避免覆盖 |
| 格式出现了变化 | Word/Excel 格式保留是尽力而为的，建议打开后微调 |
| 敏感信息是否泄露 | 最终报告 Markdown 中敏感信息已脱敏 |
