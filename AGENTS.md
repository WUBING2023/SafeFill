# Claude Code 执行指令（AGENTS.md）

## 默认行为

1. **默认使用中文回复**。
2. 用户通常不会代码，解释要清楚、步骤要具体。
3. 所有工作限定在 `D:\SafeFill\` 目录内。
4. 不得读取 `D:\SafeFill\` 之外的私人文件。
5. 不得只粘贴脚本日志。每一步都要用普通中文告诉用户：现在在做什么、发现了什么、用户下一步要做什么。

---

## 项目定位

这是一个**SafeFill**。目标是帮老师/行政人员/科研人员用已有资料半自动填写重复表格，同时严格保护隐私。核心难点不是填表逻辑，而是让用户相信资料不会泄露。

---

## 每次执行前必须读取

- `D:\SafeFill\docs\项目阶段总路线图.md`

这个文件记录了项目总进度、各阶段目标和已完成内容。开始工作前务必先读，以免因上下文压缩或记忆丢失而重复执行或跳过阶段。

## 硬性约束

### 安全约束

| 规则 | 说明 |
|------|------|
| 默认不联网 | 所有操作本地完成 |
| API 默认关闭 | 需手动配置 `api_config.json` 才能开启 |
| API 调用 | 由 `api_config.json` / 环境变量控制，用户启用 `api_first=true` + `user_accepts_api_data_risk=true` 即允许主流程调用大模型 API |
| 不得发送敏感信息 | 身份证号、手机号、地址、证件照等 |
| 不得修改原始文件 | 所有修改针对副本 |
| 不得自动修改 vault | 只有用户确认的字段才保存 |
| 不得读取项目外文件 | 仅限 `D:\SafeFill\` 内 |

### 操作约束

| 规则 | 说明 |
|------|------|
| 不得自由发挥 | 按阶段文档执行 |
| 不得扩大范围 | 一次只做一个阶段 |
| 不得安装依赖 | 除非用户明确同意 |
| 不得覆盖已有文件 | 先备份再修改 |
| 不得删除用户数据 | 用户数据目录永远不动 |
| 不得修改核心脚本 | `app\*.py` 不随意改动 |
| 不得把用户数据打包 | `vault/`、`input_forms/` 等绝不分发 |

## 用户数据目录（绝对不能动）

```
vault\              input_forms\        new_forms\
candidate_reviews\  draft_outputs\      review_results\
final_outputs\      filling_reports\    final_reports\
api_previews\       api_results\        logs\       api_logs\
```

这些目录中的任何文件都不得删除、移动、复制到项目外、或打包上传。

## 主动沟通规则

SafeFill 面向不会代码的用户。Claude Code 执行任何 SafeFill 子 Skill 时，必须像助手一样主动交流，不能只当命令执行器。

### 开始前必须说

1. 当前进入哪个 SafeFill 子 Skill。
2. 这一步的目的是什么。
3. 会读取哪些目录。
4. 会不会修改 vault、原始表格、草稿、最终文件。
5. 是否联网、是否调用 API。

### 运行后必须说

1. 用普通中文总结结果，不要让用户自己从日志里猜。
2. 如果生成网页，必须主动告诉用户浏览器地址和要点击的按钮。
3. 如果需要用户确认，必须明确说明“这里需要你确认”，不得伪造用户确认。
4. 如果发现冲突，必须展示冲突字段名、旧值、新值、推荐处理方式。
5. 如果流程可以继续，必须主动建议下一步并给出命令。
6. 如果本轮已经导出最终文件，必须提醒用户可以运行 Cleaner 预览清理临时隐私文件，但不得自动清理。

### 禁止的回答方式

- 不得只说“完成”“运行成功”“见上方日志”。
- 不得只列文件路径而不解释用途。
- 不得让用户自行判断下一步。
- 不得把“是否确认保存”“是否确认发送 API”“是否确认清理”替用户决定。

---

## 遇到以下情况时停止并报告

1. 发现真实个人隐私数据（身份证号、手机号等）——不完整展示，仅报告发现
2. 发现核心脚本缺失
3. 发现用户文件可能被覆盖
4. 发现文件保护机制失效
5. 发现可能的敏感信息泄露路径
6. 不确定某个目录是否为用户数据目录

---

## 发现真实隐私数据时的处理

1. 不要完整输出在终端或日志中。
2. 报告发现位置和字段类型，不报告值。
3. 询问用户是否需要脱敏处理。
4. 如果数据已出现在报告中，确认报告文件是否在安全位置。
